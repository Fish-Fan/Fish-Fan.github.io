  1. 并发机制底层实现原理

并发机制底层实现原理

  1. 什么是volatile
    volatile是轻量级的synchronized，它在多处理器开发中保证了共享变量的“可见性”，所谓“可见性”，就是指一个线程修改该共享变量时，其余线程能够立即读取到该共享变量最新的值。

  2. 什么是CAS操作
    循环执行某操作，直到某操作的结果和预期结果一样。

  3. CAS实现原子操作的三大问题

    • ABA问题：一个值原来是A，后来又变成了B，最后又变成了A，这时使用CAS检查时发现值没有变化，但是实际上已经发生了变化，这就是ABA问题。
      解决方法是加一个版本标记，每次对值进行操作是版本号+1，最后执行CAS检查的时候不仅要检查值，还要检查版本号是否相同。

    • 循环时间长开销大

    • 只能保证一个共享变量的原子操作

  4. happen-before原则
    要求前一个操作执行的结果必须对后一个操作可见，且前一个操作按顺序排在第二个操作之前。

  5. 顺序一致性内存模型
    同一时刻只能有一个线程访问全局内存。

  6. volatile变量的特性

    • 可见性：对一个volatile变量的读，总是能看到任意线程对volatile变量最后的写入。

    • 原子性：对任意单个volatile变量的读/写具有原子性，但是对volatile++这种复合操作没有原子性
